Add location selection inside Zone Alert activation flow.

Current behavior:
Super Admin selects a zone, then selects alert type, priority, and message.
But there is no step to choose which location(s) inside that zone are affected.

Required new behavior:

1) After selecting the zone, show the list of locations that belong to that zone
2) User must be able to select one or multiple locations
3) This location selection should appear inside the same "Activate Zone Alert" modal/sheet
4) The alert should apply only to the selected locations, not automatically to the whole zone
5) If no location is selected, do not allow activation
6) Show selected count clearly, for example:
   "3 locations selected"

UI requirements:
- Keep current modal design
- Add a new section between Zone info and Alert Type:
  "Affected Locations"
- Show all locations under the selected zone
- Use multi-select chips / cards / checkboxes
- Make it easy to select multiple locations
- Add "Select All" and "Clear All" if possible

Behavior requirements:
- Selected locations must be stored with the zone alert
- Alert monitor / map / notifications should reflect only the selected locations
- If all locations are selected, it can behave like a full-zone alert
- Existing zone alert flow must keep working

Data/model requirements:
- Add selectedLocationIds (or similar) to the zone alert payload/state
- Save these selected locations in store/state
- Pass them to any monitor / alert display / notification logic

Validation:
- Cannot activate alert unless at least one location is selected
- If user changes zone, reset selected locations to only that zone’s locations

Important:
- Do not break current emergency mode buttons
- Do not break current zone alert activation
- Do not change unrelated UI
- Keep this feature scoped to zone alert activation only

1) which files changed
2) where selected locations are stored
3) how multi-select works
4) how alert monitor/map now knows which locations are affected
