==================================================
LOCATION MODEL
==================================================

A Location is a real operational area inside a Zone.

Examples:
- OT-1
- OT-2
- OT-3
- OT-4
- WIP
- Utility
- Shipping
- CCR
- Camp
- Gas Train-1
- Gas Train-2

Each Location must have:
- id
- name
- zoneId
- zoneName
- polygon / boundary points
- expectedManpower
- supervisorUserId (optional if not assigned yet)
- backupSupervisorUserId (optional)
- isActive

If a Location already exists in the store, do NOT create a duplicate.
The map boundary must link to the same existing locationId.

==================================================
SHELTER MODEL UPDATE
==================================================

A shelter is NOT always one shelter per one location.

One shelter may serve multiple locations.

Examples:
- Shelter A -> OT-1 + OT-2
- Shelter B -> OT-3 + OT-4

Update shelter model to support:

- id
- name
- zoneId
- zoneName
- lat
- lng
- isActive
- linkedLocationIds: number[]

Important:
A shelter can be linked to multiple locations.
A location may use one shelter.
Keep the structure simple and practical.

ADMIN MAP FEATURES
==================================================

Admin must be able to manage Locations inside a Zone using the map.

Required features:

1) Add Location Boundary
- select a zone
- choose/add location
- draw polygon inside that zone
- save it to the same location record

2) Edit Location Boundary
- select location polygon
- edit points
- save

3) Delete Location Boundary
- remove polygon only or remove location entirely (choose safer behavior)
- if deleting the location would break assignments, require confirmation

4) Link Shelter to Locations
- select shelter
- choose one or more locations linked to that shelter
- save

5) View linked data
When admin taps a location boundary, show:
- location name
- zone
- expected manpower
- supervisor
- backup supervisor
- linked shelter (if any)
