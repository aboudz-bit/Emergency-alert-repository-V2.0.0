==================================================
GOAL
==================================================

Users screen is an emergency-only operational screen.

It must only become active when there is an active emergency.

Emergency conditions:
- active alert
- Shelter-in active
- Blackout active

Use:
isEmergencyActive =
  hasActiveAlert ||
  shelterIn ||
  blackout

==================================================
ROLE ACCESS RULES
==================================================

During emergency mode, the Users screen must be available to:

- Super Admin
- ECO
- Supervisor
- Backup Supervisor

Access scope rules:

1. Super Admin
- can view all users across all zones and locations

2. ECO
- must have the same visibility scope as Super Admin
- can view all users across all zones and locations

3. Supervisor
- can only view users inside the location(s) assigned to that supervisor
- example:
  Supervisor assigned to OT-1 → only sees users in OT-1

4. Backup Supervisor
- same scoped visibility logic as Supervisor
- can only view users inside the location(s) assigned to that backup supervisor

Outside emergency mode:
- Users screen must NOT operate as a live personnel monitoring screen
- do not show full user list
- do not show operational filters
- do not show grouped live personnel monitoring
- do not show emergency counts
- do not show update status controls

Instead show a clean empty state message.
