A) ALERT / ALL CLEAR

- zone alert flags
- emergency banners
- personnel emergency markers
- user response state
- safe / need_help alert-related state
- Shelter-in
- Blackout
- wind direction / eco emergency state
- any emergency overlays on the map

After All Clear:

- map returns to normal mode
- no leftover red/green emergency markers remain
- no stale emergency UI remains visible

B) SHELTER-IN / BLACKOUT MUST ACT LIKE ALERT MODE
When Shelter-in OR Blackout is activated:
- the map must switch to emergency mode
- live personnel visibility must be enabled
- personnel tracking must be active
- users must be visible on the map
- users must be able to respond with Safe / Need Help
- emergency banners must appear
- the app must behave as if an alert is active, even if no formal alert record exists

Use this concept:

isEmergencyActive =
  hasActiveAlert ||
  shelterIn ||
  blackout

All mobile logic must rely on isEmergencyActive instead of only hasActiveAlert.

This includes:
- map visibility
- personnel visibility
- alert banners
- emergency controls
- safe / need_help actions
- tracking logic
- emergency UI states

D) USER RESPONSE RESET
When a new zone alert is activated, previous mobile user response state must reset.

E) ROUTE GUARDS

- (it)

F) REGISTRATION SAFETY
Registration must NOT allow creating Super Admin directly.
Registration role must be locked to normal user-safe role only.

G) ZONE DELETE SAFETY
Prevent deleting a zone while it is active in emergency / alert state.
Either:
- block deletion, or
- do safe cascade cleanup

But do NOT leave orphaned active state.
